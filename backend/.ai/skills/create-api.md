# Create FastAPI Endpoint

## Description
Create a new FastAPI route with Pydantic schemas, validation, error handling, and structured logging. Follows ClipForge conventions: routes in `app/api/router.py`, schemas in `app/api/schemas.py`, prefix `/api` from `main.py`.

## When to Use
- Adding any new HTTP endpoint (CRUD for projects, status polling, downloads)
- Modifying an existing endpoint's request/response contract
- Adding webhook receivers or admin endpoints

## Inputs
- HTTP method (GET/POST/PUT/DELETE)
- Route path (e.g., `/projects/{project_id}`)
- Request schema (body, query params, path params)
- Response schema
- Error conditions map (404, 409, 422, 500)

## Outputs
- Pydantic schemas in `app/api/schemas.py`
- Route handler in `app/api/router.py`
- OpenAPI schema auto-generated by FastAPI

## Steps

1. **Define Pydantic schemas** in `app/api/schemas.py`. Use `BaseModel`, `Field(..., description=...)`, `field_validator` decorators for business rules. Add `model_config = {"from_attributes": True}` if mapping from ORM models. Reuse existing enums (e.g., `ProjectStatus`) rather than string literals.

2. **Add route handler** in `app/api/router.py`. Use typed decorators: `@router.get(..., response_model=X, status_code=200)`. Set `responses={404: {"model": ErrorResponse}}` for complete OpenAPI docs. Accept path params as typed function args.

3. **Validate inputs early** — framework handles Pydantic validation. For domain-level checks (state conflicts, missing resources), raise `HTTPException(status_code=..., detail=...)`. Return 404 for missing, 409 for wrong state, 400 for bad requests.

4. **Delegate to storage/services** — call `storage.get/save/update` or a use case. Never put business logic in the handler. Import from `app.state` for singletons, instantiate services for per-request use.

5. **Log entry and exit** via `structlog.get_logger().bind(endpoint=route)`. Log params, duration, result status. Never log sensitive data (API keys, tokens).

6. **Return typed response objects** — not raw dicts. FastAPI serializes the Pydantic model, applies response_model filtering, and generates accurate OpenAPI docs. Use `_to_response(project)` helper for model-to-schema mapping.

## Example

```python
# schemas.py
class CreateProjectRequest(BaseModel):
    youtube_url: str = Field(..., description="YouTube video URL")
    num_clips: int = Field(..., ge=1, le=10)
    subtitle_preset: str = "classic"

    @field_validator("num_clips")
    @classmethod
    def validate_num_clips(cls, v):
        if v not in {1, 3, 5, 10}:
            raise ValueError("num_clips must be 1, 3, 5, or 10")
        return v

# router.py
@router.post("/projects", response_model=ProjectResponse, status_code=201)
async def create_project(body: CreateProjectRequest):
    project = Project.create(body.youtube_url, body.num_clips, body.subtitle_preset)
    storage.save(project)
    return _to_response(project)
```

## Notes
- Existing routes use `APIRouter` from FastAPI, not `app` directly. Import `router` from `app.api.router`.
- All endpoints prefixed with `/api` via `app.include_router(router, prefix="/api")` in `main.py`.
- For file downloads, use `StreamingResponse` with `Content-Disposition` headers (see `/download/{project_id}` route).
- Thread safety: storage operations use `threading.Lock` internally. Long-running operations (pipeline) run in a separate thread.
- Path param `project_id: str` is auto-validated. Add `Path(pattern="...")` for UUID format constraints if needed.
