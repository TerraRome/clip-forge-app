# Render Pipeline

## Purpose

The render pipeline is the final stage of ClipForge's processing. It takes a source video, a highlight time window, a crop filter string, and an ASS subtitle file, and produces a finished 9:16 MP4 clip with GPU-accelerated encoding. All rendering is handled by `RenderService.render_clip()` in `app/services/render_service.py`.

## FFmpeg Command Assembly

```python
cmd = [
    settings.ffmpeg_path, "-y",
    "-hwaccel", "videotoolbox",          # GPU decode (macOS)
    "-ss", str(highlight.start),         # Seek to start
    "-i", video_path,                    # Input source
    "-t", str(duration),                 # Duration = end - start
    "-vf", f"{crop_filter},{ass_filter}", # Video filter chain
    "-c:v", "h264_videotoolbox",         # GPU hardware encoder (macOS)
    "-b:v", "5000k",                     # Video bitrate
    "-c:a", "aac",                       # Audio codec
    "-b:a", "128k",                      # Audio bitrate
    "-movflags", "+faststart",           # Web-optimized MP4
    "-pix_fmt", "yuv420p",              # Broad compatibility
    str(output_path),
]
```

## Filter Chain

The `-vf` argument chains two filters, applied in order:

### 1. Crop/Scale Filter (from `SmartCropService`)

Two variants depending on face detection:

**With face detected (landscape source):**
```
scale=1920:1080,crop=1080:1920:420:0
```
- Scale height to 1920 (OUTPUT_HEIGHT), width auto-calculated → e.g., 3413x1920 for 16:9 source.
- Crop 1080x1920 from the scaled frame, centered on the face's X coordinate (420px offset in this example).

**With face detected (portrait source):**
```
scale=1080:1920,crop=1080:1920:0:240
```
- Scale width to 1080 (OUTPUT_WIDTH), height auto-calculated.
- Crop 1080x1920 centered on the face's Y coordinate.

**No face detected (center fill):**
```
scale=1080:1920:force_original_aspect_ratio=increase,crop=1080:1920
```
- Scale to fill, then center-crop. Content that doesn't fit is cropped equally from both sides.

### 2. ASS Subtitle Burn-in

```
ass=/path/to/clip_01.ass
```

- Loads the ASS file generated by `SubtitleService`.
- Renders subtitles on the scaled/cropped frame at final 1080x1920 resolution.
- Text position, style, animation defined by the ASS style and dialogue events.

## Hardware Acceleration

### macOS (Current): `videotoolbox` + `h264_videotoolbox`

```
-hwaccel videotoolbox    # GPU-accelerated decode
-c:v h264_videotoolbox   # GPU-accelerated H.264 encode
```

The `videotoolbox` hwaccel handles video decoding on Apple's media engine (separate from GPU). The `h264_videotoolbox` encoder uses the hardware H.264 encoder for fast, power-efficient encoding.

**Performance:** A 60-second 1080p clip renders in ~15-30 seconds (2-4x realtime). Same clip with software encoding (`libx264`) takes ~120-180 seconds.

### Linux (Production Target): `cuvid`/`nvenc`

```
-hwaccel cuda -hwaccel_output_format cuda   # NVIDIA GPU decode
-c:v h264_nvenc                              # NVIDIA GPU encode
```

- `h264_nvenc` supports multiple presets: `p1` (fastest) through `p7` (best quality). ClipForge uses `p4` for good quality/speed balance.
- NVENC can handle 2-4 concurrent 1080p encodes on an RTX 4090.
- Bitrate: 5000k for 1080p is conservative. Could use 8000k for higher quality at the same encode cost.

## Encoding Parameters

| Parameter | Value | Rationale |
|---|---|---|
| Video codec | `h264_videotoolbox` / `h264_nvenc` | Hardware acceleration, broad device support |
| Video bitrate | 5000k | Good quality for 1080p30 vertical content |
| Audio codec | AAC | Universal compatibility |
| Audio bitrate | 128k | Clear speech, music acceptable |
| Pixel format | `yuv420p` | Required by all browsers and social media platforms |
| Faststart | `+faststart` | MOOV atom at file start for streaming |
| Keyframe interval | Auto (ffmpeg default ~250) | Acceptable for short clips |

## Subprocess Execution

All FFmpeg commands use `subprocess.run()` with:
- `capture_output=True` — captures stdout/stderr for error diagnosis.
- `text=True` — string output, not bytes.
- `timeout=600` — 10-minute hard limit per clip. A 60-second render should never exceed 5 minutes.
- `stdin=subprocess.DEVNULL` — prevents FFmpeg from reading terminal input (safer for background workers).

## Error Handling

```python
if result.returncode != 0:
    logger.error("render_failed", stderr=result.stderr)
    raise RuntimeError(f"FFmpeg render failed: {result.stderr}")
```

Common failure modes:
- **Seek failure**: `-ss` beyond video duration → ffprobe should prevent this.
- **ASS parse error**: Invalid ASS file → subtitle service generates valid ASS by construction.
- **Encoder failure**: GPU unavailable or VRAM full → no retry, manual re-render needed.
- **Disk full**: Output file unwritable → mount monitoring should catch.

## Output

- File: `{project_dir}/clips/clip_{nn:02d}.mp4` (e.g., `clips/clip_01.mp4`)
- Resolution: 1080x1920 (9:16 vertical)
- Codec: H.264 High Profile
- Container: MP4 with faststart

## Integration with Pipeline

Rendering is the final processing step per clip. After all clips render, the pipeline updates project status to `DONE`. The client downloads a ZIP archive via `GET /api/download/{id}`.

For each clip, progress advances by `35.0 / num_clips` percent (from 65% after highlight detection to 100% when all clips done).
